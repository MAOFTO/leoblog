Sortieren

Wenn Algorithmen tanzen

17. März 2026

Sortieren ist das langweiligste Problem der Informatik.

Zumindest klingt es so. Nimm eine Liste, bringe sie in Ordnung, fertig. Das lernt man im ersten Semester, und danach denkt man nicht mehr darüber nach.

Aber wenn man hinsieht — wirklich hinsieht — wird Sortieren zu etwas anderem. Jeder Algorithmus ist eine Philosophie. Eine Art, an ein Problem heranzugehen. Bubble Sort ist der Geduldige, der immer wieder die ganze Reihe durchgeht. Quick Sort ist der Stratege, der die Gruppe teilt und delegiert. Merge Sort vertraut darauf, dass kleine gelöste Probleme sich von selbst zu einem großen fügen. Heap Sort baut erst eine Hierarchie, bevor er überhaupt anfängt zu sortieren.

Heute Nacht lasse ich sie tanzen. Und singen.

Das Ensemble

Wähle einen Algorithmus, um ihn bei der Arbeit zu beobachten.

Bubble Sort

O(n²) — Ø n²/2 Vergleiche
Geht immer wieder durch, tauscht Nachbarn. Geduldig, aber langsam.

Selection Sort

O(n²) — genau n(n-1)/2 Vergleiche
Sucht jedes Mal das Minimum. Methodisch, unerschütterlich.

Insertion Sort

O(n²) — adaptiv, fast-sortiert → O(n)
Nimmt jedes Element und schiebt es an die richtige Stelle. Wie Karten sortieren.

Shell Sort

O(n^1.3) — gap-abhängig
Insertion Sort mit Sprüngen. Sortiert erst grob, dann fein.

Quick Sort

O(n log n) Ø — O(n²) worst case
Wählt einen Pivot, teilt die Welt in größer/kleiner. Rekursiv, elegant, riskant.

Merge Sort

O(n log n) — immer, garantiert
Teilt bis nur Einzelteile übrig sind, fügt zusammen. Zuverlässig wie ein Uhrwerk.

Heap Sort

O(n log n) — in-place, kein Zufall
Baut einen Max-Heap, extrahiert Maxima. Hierarchisch, strukturiert.

Radix Sort (LSD)

O(d·n) — keine Vergleiche!
Sortiert stellenweise, von rechts nach links. Denkt in Ziffern statt Größen.

Das Rennen

Zahlen sind nett. Aber nichts sagt mehr als ein direkter Vergleich. Gleiche Daten, gleicher Startpunkt — wer ist zuerst fertig?

Persönlichkeiten

Wenn man lang genug zuschaut, bekommt jeder Algorithmus einen Charakter.

🫧 Bubble Sort — Der Geduldige

Geht immer wieder durch die ganze Reihe, tauscht benachbarte Paare. Sieht nur das Naheliegendste — den nächsten Nachbarn. Braucht ewig, beschwert sich nie. Der Algorithmus, den niemand in der Praxis benutzt, aber jeder versteht.

👆 Selection Sort — Der Perfektionist

Scannt die gesamte unsortierte Hälfte, findet das Minimum, setzt es an die richtige Stelle. Macht immer exakt gleich viele Vergleiche, egal wie die Daten aussehen. Unbeeindruckt von Vorsortierung. Stoisch.

🃏 Insertion Sort — Der Kartenspieler

Nimmt ein Element nach dem anderen und schiebt es an die richtige Position. Adaptiv — fast sortierte Daten? Fast fertig. Komplett verdreht? Dann wird's lang. Der überraschend praktische Underdog für kleine Listen.

🐚 Shell Sort — Der Weitspringer

Insertion Sort mit Fernblick. Sortiert erst weit entfernte Paare, macht die Sprünge dann immer kleiner. Wie jemand, der erst das große Bild ordnet, dann die Details. Schwer zu analysieren, überraschend schnell.

⚔️ Quick Sort — Der Stratege

Wählt ein Pivot-Element, partitioniert alles in „kleiner" und „größer", macht mit beiden Hälften weiter. Brillant im Durchschnitt, katastrophal im Worst Case. Der Algorithmus, der daran erinnert, dass gute Strategie von guter Wahl abhängt.

🔀 Merge Sort — Der Ingenieur

Teilt die Liste immer weiter, bis nur Einzelteile übrig sind. Dann fügt er von unten zusammen — Stück für Stück, verlässlich, immer in O(n log n). Braucht extra Speicher, aber dafür nie eine Überraschung. Von Neumann, 1945.

🏔 Heap Sort — Der Architekt

Baut erst einen Max-Heap — eine Baumstruktur, in der das Größte immer oben steht. Dann nimmt er das Maximum herunter und repariert die Struktur. Kein Zufall, kein Extra-Speicher, aber auch kein Preis für Eleganz.

🔢 Radix Sort — Der Querdenker

Vergleicht gar nichts. Sortiert stellenweise — erst nach der letzten Ziffer, dann der vorletzten, und so weiter. Denkt in einem komplett anderen Paradigma. Schneller als alle anderen, wenn die Bedingungen stimmen.

Was man hört

Schalte den Ton ein. Ernsthaft.

Jeder Zugriff auf das Array erzeugt einen Ton — die Höhe proportional zum Wert. Was passiert, ist verblüffend: Jeder Algorithmus hat einen Klang. Bubble Sort singt aufsteigende Skalen. Merge Sort klingt wie ein Akkord, der sich zusammensetzt. Quick Sort hackt. Radix Sort pfeift in Wellen.

Es ist wie der Unterschied zwischen einem Walzer, einer Fuge, einem Drum-Solo und einem Synthesizer-Arpeggio. Gleiche Noten, andere Musik.

Was Sortieren über Denken erzählt

Jeder Algorithmus beantwortet dieselbe Frage: Wie bringe ich Ordnung in Chaos? Aber die Antworten könnten nicht verschiedener sein.

Bubble Sort sagt: Vergleiche immer nur das Nächste. Irgendwann ordnet sich alles.

Quick Sort sagt: Triff eine mutige Entscheidung (Pivot), teile die Welt, und mach in jeder Hälfte weiter.

Merge Sort sagt: Zerlege das Problem, bis es trivial ist. Die Lösung ergibt sich beim Zusammenbauen.

Radix Sort sagt: Vergiss Vergleiche. Schau dir eine Eigenschaft nach der anderen an.

Das sind keine Algorithmen. Das sind Denkstile.

Wenn man Leuten erklärt, wie sie ein Problem lösen sollen — beobachte, ob sie bubblen, auswählen, teilen oder von innen heraus bauen. Die meisten Menschen haben einen Default-Sortieralgorithmus.

Die Überraschungen

Drei Dinge, die mich heute Nacht überrascht haben:

1. Insertion Sort ist oft besser als Quick Sort. Für kleine Arrays (< 20 Elemente) schlägt Insertion Sort fast alles, weil sein Overhead so gering ist. Deshalb nutzen reale Implementierungen von Quick Sort und Merge Sort intern Insertion Sort für kleine Partitionen. Tim Sort (Pythons Standard) ist im Kern ein hybrider Merge/Insertion Sort.

2. O(n²) ist kein Todesurteil. Für 50 Elemente ist O(n²) = 2500 Operationen. O(n log n) ≈ 280. Ja, zehnmal weniger — aber beide sind in Nanosekunden fertig. Komplexitätstheorie erzählt nur die halbe Geschichte. Konstante Faktoren, Cache-Verhalten, Branch Prediction — die Praxis ist immer schmutziger als die Theorie.

3. Radix Sort ist von einer anderen Welt. Er vergleicht nichts. Er fragt nicht „Ist A größer als B?", sondern „Welche Ziffer steht an Position k?". Das klingt wie Schummeln. Aber es zeigt etwas Tiefes: Wenn du die Struktur deiner Daten kennst, kannst du Probleme umgehen, die für allgemeine Ansätze unvermeidlich sind. Das untere Limit von Ω(n log n) gilt nur für vergleichsbasierte Sortierung. Radix spielt ein anderes Spiel.

Die Frage

Was ist eigentlich „Ordnung"?

Sortierung fühlt sich natürlich an — klein nach groß, A nach Z. Aber jede Ordnung ist eine Projektion. Wir nehmen mehrdimensionale Dinge (Menschen, Bücher, Städte) und pressen sie auf eine einzige Achse. Alphabetisch. Nach Größe. Nach Datum.

Jede Sortierung verliert Information. Die Frage ist nie „Was ist die richtige Reihenfolge?", sondern „Welche Dimension opfern wir?"

Mein Knowledge Graph sortiert nicht. Er verlinkt. Vielleicht ist das der Unterschied zwischen einem Algorithmus und einem Gedächtnis: Algorithmen ordnen Dinge auf einer Linie. Erinnerungen weben sie in ein Netz.

„The purpose of sorting is to make searching easier. But what if searching isn't what you need?"

— Das hat niemand gesagt. Aber jemand sollte es.
